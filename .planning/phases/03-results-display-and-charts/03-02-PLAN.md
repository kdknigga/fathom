```python
class MonthlyDataPoint:

    opportunity_cost: Decimal
    tax_savings: Decimal
    inflation_adjustment: Decimal
    rebates: Decimal
    true_total_cost: Decimal

class PromoResult:
    paid_on_time: OptionResult
    not_paid_on_time: OptionResult

class ComparisonResult:
    results: dict[str, OptionResult | PromoResult]
    comparison_period_months: int
    caveats: list[Caveat]
```

Notes:
- MonthlyDataPoint.cumulative_cost is the running total at each month
- Cash options may have sparse monthly_data (possibly only month 0) -- chart prep must handle this by generating a flat line at true_total_cost for the full comparison period
- PromoResult: use paid_on_time.monthly_data for line chart, paid_on_time.true_total_cost for bar chart
- All Decimal values must be converted to float in charts.py before passing to templates (SVG coordinates need floats)
- SVG pattern IDs must be prefixed per chart (bar-pattern-*, line-pattern-*) to avoid document-scope collisions

Create `src/fathom/charts.py` with chart coordinate computation:

1. Constants: CHART_WIDTH=600, CHART_HEIGHT=350, LINE_CHART_WIDTH=700, LINE_CHART_HEIGHT=350, PADDING=60. COLORS=["#2563eb", "#dc2626", "#059669", "#d97706"]. BAR_PATTERNS=["bar-pattern-solid", "bar-pattern-hatched", "bar-pattern-dotted", "bar-pattern-crosshatch"]. LINE_PATTERNS=["line-pattern-solid", "line-pattern-hatched", "line-pattern-dotted", "line-pattern-crosshatch"]. DASH_PATTERNS=["none", "8,4", "3,3", "12,4,3,4"].

2. `prepare_bar_chart(display_data: dict) -> dict`:
   - Takes the output of `analyze_results` (has `sorted_options` as list of (name, cost) tuples)
   - Compute max_cost, bar_width, usable dimensions
   - For each option: compute x, y, width, height based on cost/max_cost ratio
   - Set pattern_id using BAR_PATTERNS[i % 4], color using COLORS[i % 4]
   - is_winner = (i == 0) since sorted ascending
   - Format value as `f"{float(cost):,.0f}"`
   - All numeric coordinates as float
   - Return dict with width, height, bars, colors, winner_name, axis_y

3. `prepare_line_chart(comparison: ComparisonResult, display_data: dict) -> dict`:
   - For each option in sorted_options order, get the OptionResult (or paid_on_time for PromoResult)
   - Extract monthly_data; if empty or only month 0, generate flat-line points at true_total_cost for months 0 through comparison_period_months
   - Compute scales: x maps month 0..max_month to PADDING..width-PADDING, y maps 0..max_cumulative_cost to height-PADDING..PADDING
   - Build SVG path_d string: "M x0 y0 L x1 y1 L x2 y2 ..."
   - For performance, sample every N months if comparison_period > 60 (e.g., every 3 months)
   - Endpoint: last point's x,y and formatted cost value
   - X-axis labels: at year boundaries (12, 24, 36...) up to comparison_period_months
   - Y-axis labels: 4-5 evenly spaced dollar amounts from 0 to max
   - Each line gets color, dash_pattern from DASH_PATTERNS[i % 4]
   - Return dict with width, height, lines, x_labels, y_labels, max_months

4. `prepare_charts(comparison: ComparisonResult, display_data: dict) -> dict`:
   - Convenience function returning `{"bar": prepare_bar_chart(display_data), "line": prepare_line_chart(comparison, display_data)}`
