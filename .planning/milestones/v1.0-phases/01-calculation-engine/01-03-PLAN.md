  <action>
First, remove `@pytest.mark.xfail` from tests in test_caveats.py and test_engine.py.

Implement `src/fathom/caveats.py`:
- `generate_caveats(options: list[FinancingOption], results: dict[str, OptionResult | PromoResult], settings: GlobalSettings) -> list[Caveat]`
- Iterate ALL options (not just winner):
  - **Deferred interest risk**: For PROMO_ZERO_PERCENT options with deferred_interest=True: compute required_monthly_payment = purchase_price / promo_term. Compute break_even_month. Generate Caveat with CaveatType.DEFERRED_INTEREST_RISK, severity WARNING or CRITICAL, message including the required payment and break-even month.
  - **High interest total**: When total_interest > Decimal("0.30") * purchase_price, generate Caveat with CaveatType.HIGH_INTEREST_TOTAL, severity WARNING.
- **Opportunity cost dominance** (cross-option check): Re-run True Total Cost ranking at return_rate +2% and -2%. If winner changes in either scenario, generate Caveat with CaveatType.OPPORTUNITY_COST_DOMINANCE, severity INFO, message naming which rate shift changes the winner. This requires a helper that computes TTC for a given return rate (can be a simplified version -- just recalculate opportunity cost at different rate).

Implement `src/fathom/engine.py`:
- `compare(options: list[FinancingOption], settings: GlobalSettings) -> ComparisonResult` -- the top-level orchestrator:
  1. Determine comparison_period = max(option.term_months for all options with a term; if all cash, period = 0)
  2. For each option, based on option_type:
     - CASH: total_payments = purchase_price, no interest, opportunity cost = compute_opportunity_cost(purchase_price, Decimal("0"), settings.return_rate, comparison_period)
     - TRADITIONAL_LOAN / PROMO_CASH_BACK / PROMO_PRICE_REDUCTION / CUSTOM: generate amortization schedule, compute opportunity cost on down_payment with decreasing pool, compute freed_cash_investment if term < comparison_period
     - PROMO_ZERO_PERCENT: generate dual outcomes per CONTEXT.md -- paid_on_time and not_paid_on_time scenarios. Wrap in PromoResult.
  3. For each option: if settings.inflation_enabled, compute inflation_adjustment. If settings.tax_enabled, compute tax_savings from interest schedule.
  4. Compute true_total_cost = total_payments + opportunity_cost - rebates - tax_savings +/- inflation_adjustment (for CASH: opportunity cost is lost returns, a positive cost; rebates from cash_back_amount if applicable)
  5. Populate MonthlyDataPoint.investment_balance and cumulative_cost fields
  6. Call generate_caveats with all results
  7. Return ComparisonResult

Handle rebates per option type: PROMO_CASH_BACK has cash_back_amount. PROMO_PRICE_REDUCTION effective rebate = purchase_price - discounted_price. Others have zero rebates.

All functions: pure, stateless, Decimal throughout, proper docstrings.

Run tests: `uv run pytest tests/ -v` (full suite)
