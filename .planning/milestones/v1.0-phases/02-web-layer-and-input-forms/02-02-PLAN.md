    - validate_form_data skips type-irrelevant fields (no APR validation for cash option)
    - build_domain_objects converts validated parsed data to list[FinancingOption] + GlobalSettings using Decimal (never float)
    - Return rate resolves: custom override takes precedence over preset radio
    - Inflation rate defaults to Decimal("3") when enabled but no custom rate provided
    - Tax rate defaults to Decimal("22") when enabled but no custom rate provided
  </behavior>
  <action>
Create `src/fathom/forms.py` with three public functions:

1. `parse_form_data(form_data: ImmutableMultiDict) -> dict`:
   - Extract `purchase_price` string
   - Extract options by iterating indices 0-9 looking for `options[{i}][type]` keys
   - For each option found, extract: type, label, apr, term_months, down_payment, post_promo_apr, deferred_interest (checkbox), cash_back_amount, discounted_price, custom_label
   - Extract settings: return_preset, return_rate_custom, inflation_enabled (checkbox), inflation_rate, tax_enabled (checkbox), tax_rate
   - Return structured dict: {"purchase_price": str, "options": [list of dicts], "settings": dict}

2. `validate_form_data(parsed: dict) -> dict[str, str]`:
   - Validate purchase_price: required, must be valid Decimal, must be > 0
   - For each option, validate only fields relevant to its type:
     - CASH: no extra validation
     - TRADITIONAL_LOAN: apr required (0-40%), term_months required (1-360 int), down_payment optional (0 to purchase_price)
     - PROMO_ZERO_PERCENT: term_months required, post_promo_apr optional (0-40%), down_payment optional
     - PROMO_CASH_BACK: apr required (0-40%), term_months required, cash_back_amount required (> 0), down_payment optional
     - PROMO_PRICE_REDUCTION: discounted_price required (> 0, < purchase_price), apr required (0-40%), term_months required, down_payment optional
     - CUSTOM: apr required (0-40%), term_months required, down_payment optional
   - Return rate: if custom provided, validate 0-30%; if preset, must be one of "0.04", "0.07", "0.10"
   - Error keys use dot notation: "purchase_price", "options.0.apr", "options.1.term_months", "settings.return_rate"
   - All Decimal conversions wrapped in try/except InvalidOperation

3. `build_domain_objects(parsed: dict) -> tuple[list[FinancingOption], GlobalSettings]`:
   - Convert each option dict to a FinancingOption dataclass instance
   - Use Decimal(str_value) for all money/rate fields -- NEVER float
   - Resolve return rate: custom override > preset > default 0.07
   - Convert percentage inputs to decimals for the engine (APR "5.99" -> Decimal("5.99") / 100)
   - Construct GlobalSettings with resolved values

Create `tests/test_forms.py` with tests covering all behavior items above. Write tests FIRST (red), then implement (green). Use simple dict fixtures for form data, not Flask request objects.

Module must have docstring. All public functions must have docstrings and type annotations. Use `from __future__ import annotations` if needed for forward references.

   - Pass `opt` dict with `idx` and empty `fields` (clean slate per locked decision: switching type clears all values)
   - Pass empty `errors` dict

   - Parse current form state from `hx-include` data using `parse_form_data` from forms.py

   - Return rendered `partials/option_list.html` with updated options list
   - Must preserve all existing option values (labels, types, field values) from the included form data

3. **POST /partials/option/&lt;int:idx&gt;/remove** -- Remove option endpoint:
   - Parse current form state from `hx-include` data

   - Re-index remaining options sequentially (0, 1, 2...) to avoid gaps
   - Return rendered `partials/option_list.html` with updated options list
   - Must preserve all remaining option values
