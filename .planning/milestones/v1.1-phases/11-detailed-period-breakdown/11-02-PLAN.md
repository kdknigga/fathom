  <action>
    **1. Add helper functions to results.py:**

    `build_detailed_breakdown(options_data, comparison_period_months, settings, granularity="monthly")`:
    - Takes options_data from analyze_results, returns a list of per-option detailed data dicts
    - Each dict has: `name`, `is_promo`, `is_winner`, `rows` (list of period row dicts), `totals` (column sums)
    - Each row dict has: `period` (month number or "Year N"), `payment`, `interest_portion`, `principal_portion`, `opportunity_cost`, `inflation_adjustment`, `tax_savings`, `cumulative_true_total_cost`
    - `cumulative_true_total_cost` = running sum of (payment + opp_cost - tax_savings + inflation_adj) across periods. Use the cumulative approach: for each month, add that month's net cost factors to the running total.
    - For promo options, build two sets of rows (paid_on_time and not_paid_on_time monthly_data)
    - If `granularity="annual"`, group months into 12-month chunks summing all fields except cumulative (use last month's cumulative)

    `aggregate_annual(monthly_rows)`:
    - Groups row dicts by year (0-11 = Year 1, 12-23 = Year 2, etc.)
    - Sums payment, interest, principal, opp_cost, inflation, tax per group
    - cumulative_true_total_cost = last row's value in each group
    - Returns list of annual row dicts with `period` as "Year 1", "Year 2", etc.

    `build_compare_data(options_data, comparison_period_months, granularity="monthly")`:
    - Returns list of row dicts for compare tab
    - Each row: `period`, then for each option: `payment` and `cumulative_true_total_cost`
    - For promo options, show paid_on_time values
    - Structure: `{"period": 1, "options": [{"name": "Cash", "payment": ..., "cumulative": ...}, ...]}`

    **1. Create detail_toggle.js:**

    IIFE with "use strict". Three concerns:

    a) **Column toggle state management:**
    - `var hiddenCols = {};` -- maps CSS class name to boolean (true = hidden)
    - On checkbox `change` event (delegated on document): read `data-column`, update `hiddenCols`, call `applyHidden()`
    - `applyHidden(container)`: iterate `hiddenCols`, find all `.col-{name}` elements in container, set `display: none` or `""` based on state
    - On `htmx:afterSwap` for `#detail-panel`: call `applyHidden(evt.detail.target)` to reapply state to new DOM

    b) **Tab keyboard navigation (ARIA tablist pattern):**
    - Delegated `keydown` on `.detail-tabs` container
    - Arrow Left/Right: move focus to prev/next `[role="tab"]`
    - Home/End: move focus to first/last tab
    - On tab click: update `aria-selected` on all tabs, set clicked to `true`, update `#detail-panel` `aria-labelledby`

    c) **Granularity toggle:**
    - On change of the monthly/annual radio/select, update a `var currentGranularity` variable
    - Re-fetch the current active tab with `?granularity=` appended
    - Use `htmx.ajax()` to trigger the re-fetch programmatically, or update `hx-vals` on tab buttons

    **2. Add CSS to style.css:**

    a) **Tab styles (underline tabs, Pico-native feel):**
