  <name>Task 1: Playwright browser verification for detailed period breakdown</name>
  <files>tests/playwright_verify.py</files>
  <action>
    Use Playwright MCP to automate ALL browser-based verification. Start the app with `uv run fathom` (or use the test server if conftest provides one), then run through these verification scenarios:

    **Setup:** Navigate to the app. Fill in a comparison with at least 2 options (e.g., Cash at $10,000 and a Traditional Loan at 6% APR, 36 months). Enable opportunity cost (7% return rate), inflation (3%), and tax savings (22%). Submit the form.

    **DETAIL-01: Per-period data renders**
    - Scroll to "Detailed Period Breakdown" section
    - Verify a table exists with column headers: Period, Payment, Interest, Principal, Opportunity Cost, Inflation Adj., Tax Savings, Cumulative True Total Cost
    - Verify the table has the expected number of rows (36 for a 36-month loan)
    - Verify non-zero values appear in the Opportunity Cost column
    - Verify the totals row (tfoot) exists and has values

    **DETAIL-02: Tab switching**
    - Verify tabs exist for each configured option (e.g., "Pay in Full", "Traditional Loan")
    - Click the second tab
    - Verify the table content changes (different values in cells)
    - Verify the clicked tab shows `aria-selected="true"` and the first tab shows `aria-selected="false"`

    **DETAIL-03: Compare tab**
    - Click the "Compare" tab
    - Verify the table shows columns for each option with Payment and Cumulative True Total Cost
    - Verify the table has the same number of period rows
    - Verify each option's name appears as a column group header

    **DETAIL-04: Column toggles**
    - Switch back to an option tab
    - Uncheck the "Interest" checkbox
    - Verify all cells with class `col-interest` are hidden (display: none or not visible)
    - Switch to another tab
    - Verify the Interest column is still hidden on the new tab (state persists)
    - Re-check the "Interest" checkbox
    - Verify the column reappears

    **DETAIL-05: Accessibility**
    - Verify the tab container has `role="tablist"`
    - Verify each tab button has `role="tab"`
    - Verify the panel has `role="tabpanel"`
    - Verify table `<th>` elements have `scope="col"` attribute
    - Use keyboard: Tab to the first tab, press Right arrow, verify focus moves to next tab
    - Press Enter on the focused tab, verify it activates (aria-selected becomes true)

    **Granularity toggle:**
    - Switch to annual view
    - Verify the row count decreases (e.g., 36 months becomes 3 years)
    - Verify period labels show "Year 1", "Year 2", etc.
    - Switch back to monthly view, verify row count returns to original

    **Sticky totals (visual check):**
    - If the table is long enough, scroll within the table container
    - Take a screenshot to verify the totals row is visible at the bottom while scrolled

    Add appropriate assertions and screenshots to `tests/playwright_verify.py` following the existing pattern in that file.
