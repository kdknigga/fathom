1. In base.html line 2: change `<html lang="en" data-theme="light">` to `<html lang="en">`. This single change enables Pico CSS built-in dark mode auto-detection.

2. In style.css, refactor all hardcoded hex colors to use CSS custom properties. Add a `:root` block at the top defining light-mode defaults, then add a `@media (prefers-color-scheme: dark) { :root { ... } }` block with dark variants.

Light-mode defaults (`:root` block):
```
--caveat-warning-bg: #fef3c7
--caveat-warning-text: #92400e
--caveat-warning-border: #f59e0b
--caveat-critical-bg: #fee2e2
--caveat-critical-text: #991b1b
--caveat-critical-border: #ef4444
--caveat-info-bg: #dbeafe
--caveat-info-text: #1e40af
--caveat-info-border: #3b82f6
--winner-star-color: #f59e0b
--winner-col-bg: rgba(59, 130, 246, 0.06)
```

Dark-mode overrides (`@media (prefers-color-scheme: dark) { :root { ... } }`):
- Caveat backgrounds: dark desaturated versions (e.g., warning: #422006, critical: #450a0a, info: #1e3a5f)
- Caveat text: lighter versions for readability (e.g., warning: #fbbf24, critical: #fca5a5, info: #93c5fd)
- Caveat borders: slightly brighter than light mode to pop against dark bg
- Winner star: keep #f59e0b or slightly brighter #fbbf24
- Winner column: increase opacity (e.g., rgba(59, 130, 246, 0.15)) so it's visible against dark bg

3. Replace all hardcoded hex values in existing CSS rules with the new `var()` references:
- `.caveat-warning { background: var(--caveat-warning-bg); color: var(--caveat-warning-text); border-left-color: var(--caveat-warning-border); }`
- Same pattern for `.caveat-critical` and `.caveat-info`
- `.winner-star { color: var(--winner-star-color); }` (both occurrences)
- `.winner-col { background: var(--winner-col-bg); }`

4. For the breakdown table sticky `.row-label` column: it uses `background: var(--pico-background-color)` which Pico already handles in dark mode. No change needed.

Do NOT redefine any `--pico-*` variables that Pico already handles (background, text, border colors, form styles). Only define variables for colors Pico does not cover (caveats, winner elements, chart colors).

Per user decision: WCAG AA contrast is required for light mode only. Dark mode should be legible but not held to strict 4.5:1 ratios.
