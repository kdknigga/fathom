  <action>
1. Add chart CSS custom properties to the existing `:root` block in style.css (created by plan 01):
```
--chart-grid: #e5e7eb;
--chart-label: #6b7280;
--chart-bar-label: #374151;
--chart-bg: #ffffff;
```

2. Add dark overrides to the existing `@media (prefers-color-scheme: dark) { :root { ... } }` block:
```
--chart-grid: #374151;
--chart-label: #9ca3af;
--chart-bar-label: #d1d5db;
--chart-bg: var(--pico-background-color);
```

3. For series colors (blue, red, green, orange from charts.py COLORS array): The Python code passes these as `bar.color` and `line.color` to templates. Since we cannot change Python-provided colors via CSS variables in inline SVG attributes, add CSS rules that override SVG fill/stroke for dark mode. The approach:
   - In the SVG templates, add a CSS class to color-bearing elements (e.g., `class="chart-series"` with `data-series="0"` etc.)
   - OR define `--chart-series-0` through `--chart-series-3` CSS variables and use them in templates instead of `{{ bar.color }}` / `{{ line.color }}`
   - Use brighter/lighter variants in dark mode: e.g., #60a5fa (blue), #f87171 (red), #34d399 (green), #fbbf24 (orange)

   Recommended approach: Define CSS variables `--chart-series-0: #2563eb` etc. in `:root`, override in dark `@media`, and reference in templates as `fill="var(--chart-series-{{ loop.index0 }})"`. This requires the template loop index to match the Python color index (which it does - both iterate options in order).

4. In bar_chart.html:
   - Pattern `fill="white"` (4 occurrences in pattern defs) -> `fill="var(--chart-bg, white)"`
   - Y-axis baseline `stroke="#e5e7eb"` -> `stroke="var(--chart-grid)"`
   - Bar name labels `fill="#374151"` -> `fill="var(--chart-bar-label)"`
   - Value labels `fill="{{ bar.color }}"` -> `fill="var(--chart-series-{{ loop.index0 }})"`
   - Bar stroke `stroke="{{ bar.color }}"` -> `stroke="var(--chart-series-{{ loop.index0 }})"`
   - Bar `style="color: {{ bar.color }}"` -> `style="color: var(--chart-series-{{ loop.index0 }})"`

5. In line_chart.html:
   - Grid lines `stroke="#e5e7eb"` (multiple) -> `stroke="var(--chart-grid)"`
   - Axis labels `fill="#6b7280"` (multiple) -> `fill="var(--chart-label)"`
   - Line stroke `stroke="{{ line.color }}"` -> `stroke="var(--chart-series-{{ loop.index0 }})"`
   - Endpoint label `fill="{{ line.color }}"` -> `fill="var(--chart-series-{{ loop.index0 }})"`

Note: The Jinja2 loop variable is `loop.index0` (0-based). Verify the loop variable name matches the template's `{% for bar in ... %}` / `{% for line in ... %}` iteration.
