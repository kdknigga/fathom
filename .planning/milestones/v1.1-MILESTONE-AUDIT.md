1. **Dark mode + All UI (7→all):** CSS custom properties in `style.css` consumed by SVG chart templates, tooltip CSS, detail table tabs — all via `var()` references with dark overrides
2. **Comma formatting + Tax brackets (8→9):** `|comma` Jinja filter used in `global_settings.html` for bracket income ranges; `comma_format` extended to handle `int/float` for bracket data
3. **Comma formatting + Detailed breakdown (8→11):** Server-side `_clean_monetary` ensures Decimal values are clean; detail tables use `"{:,.2f}"` format strings for read-only display

5. **Per-period data pipeline (11):** `MonthlyDataPoint` extended fields → engine builders populate → `build_detailed_breakdown` → HTMX-driven tab UI
6. **JS event delegation (8,9,11):** `formatting.js`, `tooltips.js`, `detail_toggle.js` all use event delegation — HTMX-swapped content inherits behavior automatically; no conflicts between selectors
7. **Phase 12 refactoring safety:** All refactored functions preserve public API signatures; 241 tests confirm zero breakage
